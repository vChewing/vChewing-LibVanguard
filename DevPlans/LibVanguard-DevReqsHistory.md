| Phase 11 | ⚠️ **已作廢回滾** TextMapTrie / FactoryTextMapLexicon DFD（Direct File Disk）硬碟直讀模式。三倉庫同步導入 `StorageMode` 列舉，區分 `.inMemory(Data)` 與 `.directFileAccess(fileHandle:readingKeys:)`（LibVanguard 另含 `materializedKeys`）兩種儲存策略。DFD 模式的 `init(fileURL:)` 以 `.mappedIfSafe` 暫時 mmap 原始檔案建立全部索引（PRAGMA 邊界、行偏移、KEY_LINE_MAP、反查表），同時物化 reading key 字串陣列，完成後釋放 mmap 僅保留 `FileHandle`；runtime 查詢透過 POSIX `pread(fileHandle.fileDescriptor, &buffer, length, off_t(offset))` 原子讀取 VALUES 行，執行緒安全且無需額外鎖。穩態記憶體由原先 ~15 MB raw `Data` 常駐降至 ~3 MB 索引（節省約 12 MB）。LibVanguard `TextMapTrie`：`extractValueLine()` / `parseNodeEntries()` / `parsedReadings()` 依 `storageMode` 分派；`VanguardTrieIO` 新增 `loadFromTextMapDFD(url:)` 靜態方法。vChewing-macOS `FactoryTextMapLexicon`：`resolveKey` / `binarySearchIndex` / `parsedEntries` / `parsedReadings` 改為 index-based API，DFD 二分搜尋以 `compareUTF8Arrays()` 對物化鍵陣列操作；`connectFactoryDictionary` 的同步與非同步路徑皆改用 `FactoryTextMapLexicon(fileURL:)` 建構 DFD 實例；`connectToTestFactoryDictionary` 維持 in-memory 模式不變。`FileHandle` 關閉統一使用 `closeFile()`（相容 macOS 10.15 以下）。vChewing-OSX-Legacy 同步移植相同變更。驗證結果為 LibVanguard 92 項、LangModelAssembly 51 項、Typewriter 28 項測試全數通過，Legacy `make debug-core` 建置成功。**但實機測試發現記憶體佔用量反而飆升 20 MB ~ 30 MB，手術全數撤銷。** 失敗原因：(1) `pread()` 每次查詢產生短暫 `Data` buffer，malloc free-list 不立即歸還 OS，heap 持續膨脹（heap fragmentation）；(2) DFD binary search 需先物化全部讀音鍵為 Swift `String` 陣列（~50,000+ 物件 + ARC 開銷），original in-memory 模式以 byte range 直指 `Data` 幾乎零額外 allocation；(3) `Data(contentsOf:)` 底層走 mmap 時 OS 可標記頁面為 purgeable/discardable 並在記憶體壓力時主動回收，`pread()` 碎片化 heap 無此優勢。結論：「大塊單一連續資料 + 惰性 byte-range 解析」已是最佳平衡，DFD 化在此場景弊大於利，不建議再次嘗試（除非改以 OS-level `mmap` + arena 管理查詢 buffer）。 |
| Phase 12 | LMCassette（CIN2 磁帶模組）連續記憶體最佳化。仿照 `FactoryTextMapLexicon` 模式，將 `charDefMap`、`charDefWildcardMap`、`symbolDefMap`、`reverseLookupMap` 四個大型 `[String: [String]]` Dictionary 替換為 `CassetteSortedMap`（contiguous `Data` blob + sorted `[CassetteMapEntry]` byte-range index + `[CassetteByteRange]` flat values array + binary search）；`octagramMap: [String: Int]` 替換為 `CassetteOctagramMap`；`octagramDividedMap: [String: (Int, String)]` 替換為 `CassetteOctagramDividedMap`。實測發現初版仍有偏高的載入期 HEAP 峰值後，於 2026-04-13 進行第二輪修正：`build(from:)` 改為直接接受 grouped Dictionary；`buildWildcard(from:wildcard:)` 與 `buildReverseLookup(charDefs:symbolDefs:)` 改為以 sorted lightweight prototypes 分組建構，移除 `tmpReverseLookup`、`tmpCharDefWildcard`、`tmpDict.map { ... }` 與 sort comparator 內的 `Array(rawData[range])` allocation。修改範圍跨 `vChewing-macOS` 與 `vChewing-OSX-Legacy` 兩倉庫。驗證結果為 LangModelAssembly 51 項、Typewriter 28 項、MainAssembly 2 項 Cassette E2E 測試全數通過；Legacy `make debug-core` 建置成功。 |
